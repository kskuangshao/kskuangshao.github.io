hadoop伪分布式安装部署

hadoop伪分布式安装部署

1.wget下载(这种方式快)

wget http://archive.cloudera.com/cdh5/cdh/5/hive-1.1.0-cdh5.16.2.tar.gz

2.关于目录

1.创建文件目录

[ks@JD ~]$mkdir app software sourcecode log data tmp lib

a.解压到家目录/app

[ks@JD software]$ tar -xzvf hadoop-2.6.0-cdh5.16.2.tar.gz -C ~/app/

b.做软连接（方便版本更新）（注意前后）

[ks@JD app]$ln -s hive-1.1.0-cdh5.16.2 hive

3.Java环境

/usr/java #位置必须是这样

4.hadoop环境变量

[ks@JD ~]$ vi .bashrc
export HADOOP_HOME=/home/ks/app/hadoop
export HIVE_HOME=/home/ks/app/hive
export PATH=${HADOOP_HOME}/bin:${HADOOP_HOME}/sbin:${HIVE_HOME}/bin:$PATH
[ks@JD ~]$ source .bashrc 
[ks@JD ~]$ which hdfs
[ks@JD ~]$ which hive

https://hadoop.apache.org去找single node cluster单节点伪分布式资料

5.配置ssh信任关系

家目录下输入
  $ ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
  $ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
  $ chmod 0600 ~/.ssh/authorized_keys

6.etl/hadoop下配置文件

core-site.xml

   <property>
       <name>fs.defaultFS</name>
       <value>hdfs://focus:9000</value>
   </property>
   <!--hadoop的tmp目录30天清理一次-->
<property>
       <name>hadoop.tmp.dir</name>
       <value>/home/ks/tmp</value>
   </property>

hdfs-site.xml

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <!--下面两个参数snn以focus这个localhost启动，不是0.0.0.0启动-->

    <property>
        <name>dfs.namenode.secondary.http-address</name>
        <value>focus:9868</value>
    </property>

    <property>
        <name>dfs.namenode.secondary.https-address</name>
        <value>focus:9869</value>
    </property>
</configuration>

mapred-site.xml

<property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
</property>

yarn-site.xml

    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
<!--以防被挖矿-->	
	<property>
        <name>yarn.resourcemanager.webapp.address</name>
        <value>focus:18088</value>
   	 </property>

hadoop-env.sh

java环境变量export

7.格式化目录

hdfs namenode -format (看是否出现successfully)

8.启动

start-dfs.sh
start-all

9.查看hdfs

hdfs dfs -ls /
